## Task

### Croatian
```
Napisati algoritam za CRCW/EREW PRAM računalo
koji će za zadano polje P[]
provjeriti ima li u polju elemenata jednakih vrijednosti (duplikata).
Za polje od n elemenata na raspolaganju je n procesora.
Rezultat mora biti zapisan u jednoj izlaznoj varijabli.
```

### English
```
Write an algorithm for a CRCW/EREW PRAM computer
that will, for a given array P[],
check whether there are elements with equal values in the array (duplicates).
For an array of n elements, n processors are available.
The result must be written in one output variable.
```

**Requirement: O(n)**


## My solution

### CRCW
```
has_duplicates = false;

parallel (i = 0 ; i < n - 1 ; i++) { // O(n)
    for (j = i + 1 ; j < n ; j++) {
        if (P[i] == P[j]) {
            has_duplicates = true;
        }
    }
}

return has_duplicates;
```
_We could also use a hashmap if we know that collisions are handled nicely.<br>
However, that would still be O(n) because all element hashes could collide._

### EREW
```
C[]; // We'll make a copy of P
R[]; // Results

parallel (i = 0 ; i < n ; i++) { // O(1)
    C[i] = P[i];
    R[i] = false;
}

parallel (i = 0 ; i < n - 1 ; i++) { // O(n)
    for (j = i + 1 ; j < n ; j++) {
        if (P[i] == C[j]) {
            R[i] = true;
            break;
        }
    }
}

return or_reduce(R); // O(log(n))
```
_Because of PRAM's lock-step, we never have multiple processors accessing the same memory._
